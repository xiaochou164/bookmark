# Raindrop 前端对齐截图对比流程（Web）

## 目标

- 用固定页面、固定尺寸、固定数据状态做截图对比，追踪 UI 对齐进度与回归。

## 1. 截图范围

- 首页工作台（行列表）
- 首页工作台（卡片视图）
- 首页工作台（标题视图）
- 首页工作台（看板视图）
- 详情面板（查看态）
- 详情面板（编辑态）
- 预览弹窗（网页模式）
- 预览弹窗（阅读模式）
- 设置页（账号与安全）
- 设置页（共享与公开）
- 设置页（产品能力）
- 登录页
- 公开分享页

## 2. 固定前置条件

- 使用同一组测试数据（含封面、有/无元数据、失败抓取、高亮注释、标签较多条目）。
- 浏览器缓存清理后再截图（避免历史样式缓存）。
- 浏览器缩放 100%。
- 使用固定主题（当前仅浅色）。

## 3. 建议 viewport

- 桌面：`1440 x 900`
- 笔记本：`1280 x 800`
- 手机：`390 x 844`

## 4. 截图命名规范

- `home-list-1440x900.png`
- `home-card-1440x900.png`
- `detail-readonly-1440x900.png`
- `preview-reader-1440x900.png`
- `settings-account-1440x900.png`
- `login-390x844.png`

## 5. 对比维度（每张图）

- 信息架构：模块位置、分组层级是否接近目标
- 视觉密度：间距、字体层级、控件尺寸
- 交互态：hover/active/selected/focus 是否一致
- 状态反馈：加载/失败/空态是否完整
- 响应式：换行是否自然、控件是否溢出

## 6. 判定规则

- `通过`：布局和信息层级稳定、无明显错位/溢出
- `需优化`：视觉密度或细节与目标差距明显，但功能可用
- `失败`：错位、遮挡、溢出、交互状态异常或功能阻断

## 7. 回归触发时机

- 任何一轮 `RA-UI-*` 样式/交互改动后
- 对话框、预览、设置页结构重构后
- 重大 CSS token/排版调整后
