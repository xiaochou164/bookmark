# Rainboard UI/UX 全项目审计与修改 TODOLIST

审计日期：2026-07-04

专项测试清单：`docs/UI_UX_TEST_CHECKLIST.md`

## 审计范围与结论

覆盖范围：Web 主工作台、登录页、设置页、同步插件页、公开分享页、Chrome 扩展 Popup/Options，以及桌面、平板、手机断点。

主要证据：

- `public/styles.css` 已超过 7,000 行，存在大量完全重复的核心选择器；`.toolbar`、`.view-header`、`.tree-item`、`.bookmark-row` 等被 3～5 处重复定义。
- 样式中有 31 组媒体查询、304 处固定像素尺寸、28 处 `!important`，断点集中在 `640/720/920/980/1080/1180/1280/1480px`，缺少统一矩阵。
- 主工作台 HTML 有 186 个按钮、10 个 Dialog；`public/app.mjs` 超过 9,000 行，UI 状态和渲染逻辑高度耦合。
- 设置页和插件页仍有大量 `<pre>` 原始 JSON 输出；设置页存在 5 处原生 `confirm`，主应用启动失败仍使用阻塞式 `alert`。
- Chrome 扩展 Popup/Options 使用独立内联 CSS，颜色、间距、按钮高度和响应式策略均未复用 Web 设计系统。

优先级：`P0` 阻碍正常使用或持续制造冲突；`P1` 明显影响效率和一致性；`P2` 体验完善。

状态：`TODO / DOING / DONE / BLOCKED`。每项完成后需补桌面 `1440×900`、笔记本 `1280×800`、平板 `768×1024`、手机 `390×844` 的截图或交互记录。

- [ ] `UI-AUD-001` 拆分 `public/styles.css`：按 `tokens/base/layout/workbench/components/dialogs/settings/public/responsive` 分文件，入口文件只负责导入；验收为核心选择器不再跨 3 个以上文件重复定义。
- [ ] `UI-AUD-002` 清理重复与失效规则：逐项合并 `.toolbar/.view-header/.tree/.bookmark-row/.detail/.context-menu` 的多代覆盖，移除无消费者样式；验收为完全重复选择器数量下降至少 80%。
- [ ] `UI-AUD-003` 建立统一尺寸 token：控件高度限定为 `28/32/36/40px` 四档，统一圆角、间距、侧栏宽度、详情栏宽度和图标尺寸；禁止页面自行发明相邻尺寸。
- [ ] `UI-AUD-004` 收敛响应式断点为 `640/920/1180/1280px` 四档，移除 `720/980/1080/1480px` 的局部补丁或并入最近断点；输出断点行为表。
- [ ] `UI-AUD-005` 建立页面视觉回归：登录、主列表、卡片、详情抽屉、设置、插件、公开页各保存四档基线截图，并在发布清单中强制检查。
- [ ] `UI-AUD-006` 建立统一层级规范：整理 sticky header、popover、menu、drawer、dialog、toast 的 z-index；验收覆盖“菜单位于抽屉内”“Dialog 上再开确认”“移动端遮罩”三种场景。
- [ ] `UI-AUD-007` 全局横向溢出审计：不得依赖 `html { overflow-x:hidden }` 掩盖问题；在四档宽度下 `documentElement.scrollWidth <= clientWidth`。

## P0：主工作台核心可用性

- [ ] `UI-AUD-101` 重排顶部工具栏：第一层只保留搜索、添加和当前视图核心操作；排序、视图、导出等收进集合标题栏或更多菜单，避免 920～1280px 反复换行。
- [ ] `UI-AUD-102` 明确桌面 Split View：侧栏、主区、详情栏使用一套网格规则，详情打开时主列表不得被遮住或突然缩成不可读宽度。
- [ ] `UI-AUD-103` 改造平板/手机导航：侧栏和详情统一为 overlay drawer，提供明确的打开入口、返回按钮、遮罩关闭和焦点回收，不再把侧栏排到主内容下方。
- [ ] `UI-AUD-104` 降低书签行操作密度：默认只显示 3 个高频操作，其余进入更多菜单；键盘焦点和触屏场景必须能发现操作，不能仅依赖 hover。
- [ ] `UI-AUD-105` 统一四种视图的信息层级：标题、域名、集合、标签、时间在列表/卡片/标题/看板中使用一致语义；明确各视图允许隐藏的次要字段。
- [ ] `UI-AUD-106` 重构详情栏头部：导航、模式、预览、更多、关闭分成稳定的两行或分组，不允许工具随标题长度挤压；窄详情栏下仍完整可操作。
- [ ] `UI-AUD-107` 精简详情栏区块：默认只展开基本信息和常用状态；抓取、AI、高亮、相关推荐按使用频率折叠，避免首屏成为长表单。
- [ ] `UI-AUD-108` 统一 10 个 Dialog 的结构：标题、说明、主体滚动区、错误区、主次操作栏一致；最大高度不超过视口，主操作始终可见。
- [x] `UI-AUD-109` 移除主应用启动失败的原生 `alert`，改为可恢复错误页，提供“重试加载”“重新登录”“复制诊断信息”。
- [ ] `UI-AUD-110` 统一 loading/empty/error/skeleton：列表、详情、预览、搜索、任务结果使用同一状态组件，避免空白区或纯文本错误。

1. Sprint A：`UI-AUD-001~007`，先停止 CSS 继续互相覆盖。
2. Sprint B：`UI-AUD-101~110`，收口主工作台与移动端核心路径。
3. Sprint C：`UI-AUD-201~208`，重做设置和任务结果展示。
4. Sprint D：`UI-AUD-301~505`，统一外围页面、扩展和可访问性。
5. Sprint E：`UI-AUD-601~605`，模块化、主题和性能治理。

## 完成定义（DoD）

- 对应任务有修改前/后截图或录屏。
- 四档视口无横向溢出、遮挡、不可达操作或双滚动。
- 键盘可完成该功能主流程，焦点不会丢失。
- loading/empty/error/success 均有明确状态。
- `npm run cf:check`、`npm run cf:smoke` 和远端 smoke 通过。
- 若修改核心布局或组件，必须更新视觉回归基线和前端回归清单。
