## 2026-07-05 Raindrop `/my/0` 对照优化方案与本轮实施

对照原则：Raindrop 的主工作台优势不是装饰更多，而是把长期整理书签需要的操作压到稳定位置：左侧集合树、顶部全局搜索、中间高密度列表、右侧详情/预览，行内操作默认轻量，更多能力进入菜单或详情。

### 2026-07-05 继续对照发现与实施

本轮继续按 Raindrop `/my/0` 的工作台密度审查，发现仍有三类设计不合理点：

- 顶栏仍有可见文字型工具按钮（通知、询问、刷新），会在 920～1280px 断点抢占搜索空间，首屏水合前后也存在视觉跳变。
- 列表行右侧操作默认常驻，信息列最小宽度偏大，导致标题/域名区域在详情面板打开或中宽视口下过早被挤压。
- 右侧详情未选中空态仍像独立卡片，视觉重量高于 Raindrop 的轻提示侧栏状态，容易分散对列表的注意力。

真实 Raindrop `/my/0` 截图确认的参照点：左侧集合树为浅灰固定栏并优先展示计数，顶部以搜索和少量图标/主按钮为核心，中间列表采用缩略图 + 标题 + 域名/集合/日期的高密度行，未打开详情时右侧保留安静空白区域而不是额外信息卡。

开启 Chrome AppleScript DOM 读取后的实测数值：侧栏宽 `300px`、背景 `rgb(246,245,244)`、侧栏行高 `32px`；顶部主 header `48px`，视图 header `48px`；列表行 `80px`，标题 `15px/500`，元信息 `13px` 且使用 `rgb(128,128,128)`。

本轮已实施：

- [x] 顶栏按钮进一步图标化：通知、询问 AI、刷新在初始 DOM 与 JS 水合后均使用屏幕阅读器文本，避免首屏文字闪现和搜索区被挤压。
- [x] AI 模式徽标从主顶栏移除可见占位，保留 AI 入口本身，减少非 Raindrop 核心能力对主搜索工具条的干扰。
- [x] 列表行右侧信息列改为 `clamp(168px, 22vw, 300px)`，并让行内动作在桌面默认隐藏、hover/focus/active/selected 时显示，保持密度且不引发布局抖动。
- [x] 详情空态去卡片化，改为居中轻提示，降低未选中状态下右栏视觉噪音。
- [x] 默认隐藏快捷键教学条，保留快捷键能力但不常驻占用列表上方空间，更接近 Raindrop 的纯内容列表。
- [x] “已加载全部”和单页分页器默认隐藏，仅在确有下一页/多页时显示底部控制，减少列表底部工具噪音。
- [x] 桌面视图头部提升“导出书签”入口，匹配 Raindrop 右侧显式导出操作；移动端仍收进更多菜单。
- [x] 移动端列表改回“选择框 + 小缩略图 + 文本信息”的紧凑行结构，隐藏行内动作区，避免窄屏出现竖排文字按钮或封面占满整行。
- [x] 侧栏底部 Rainbow 状态卡默认隐藏，避免和 Raindrop 的集合/过滤优先侧栏模型冲突。
- [x] 将桌面侧栏宽度与外层网格改为 Raindrop 实测的 `300px`，移除桌面 shell 外层 gap/padding，并同步侧栏浅灰背景。
- [x] 将列表行高度、标题字号/字重、元信息字号/颜色同步到 Raindrop 实测 DOM 数值。

### 2026-07-05 设置页与书签编辑/详情页对齐补充

用户追问后继续覆盖设置页和书签编辑/详情页。Raindrop 设置页实测为 `/settings/app`：左侧仍是 `300px` 浅灰侧栏，右侧白色主画布，顶部标题为“应用设置”，内容不是卡片宫格，而是纵向的设置项与轻分隔表单。

本轮针对这两页发现的不合理点：

- 设置页旧版把侧栏、页头、每个设置区都做成浮卡，和 Raindrop 的同一应用壳不一致，视觉上更像后台管理台。
- 设置页导航用“账号与安全/共享与公开/产品能力/任务与运维”等本地后台术语，和 Raindrop 的“应用/帐户/订阅/导入/整合方式/备份/2FA”语气距离较大。
- 书签详情页右侧面板分区卡片过多，输入框边框密度偏重，底部动作栏与顶部工具条重复，编辑/查看状态没有 Raindrop 属性面板那种轻量感。

本轮已实施：

- [x] 设置页改为 Raindrop 式固定 `300px` 左侧栏，背景 `rgb(246,245,244)`，右侧主区白底无浮卡、无外层圆角。
- [x] 设置页页头改为“应用设置”，导航文案收敛为“帐户 / 订阅 / 整合方式 / 备份 / AI”，减少后台感。
- [x] 设置页内容从双列卡片网格改为纵向表单行：区块用细分隔线，标签左、控件右，按钮与状态输出降噪。
- [x] 设置页移动端移除外层页边距，保持同一应用壳的贴边布局。
- [x] 书签详情/编辑页改为轻量右侧 inspector：顶部工具条 `48px` 级别，按钮 `28px`，摘要卡降到 8px 圆角。
- [x] 详情分区去卡片化，改为白底细分隔线表单；输入框使用底线式边界，查看态避免禁用表单的压迫感。
- [x] 详情底部操作条改为贴底轻分隔，和顶部打开/预览/删除等工具动作形成主次分工。

- [x] 顶部工具栏降噪：刷新改为图标按钮，排序/视图/更多补齐 SVG 图标，通知中心改为铃铛图标，避免中文文字按钮在 920～1280px 断点抢占搜索空间。
- [x] 列表行操作降噪：桌面列表行保留“新标签页打开 / 收藏 / 更多操作”，移除与点击行重复的“预览模式”按钮；卡片仍保留显式预览入口。
- [x] 详情面板主次动作收口：只读态隐藏底部重复操作栏，编辑态保留保存/取消/编辑入口；打开、预览、删除等已在详情头部工具区承载。
- [x] 详情只读态阅读化：查看模式下基本信息从禁用表单改为轻量字段展示，编辑模式再恢复输入框，降低右侧面板的表单压迫感。
- [x] 卡片视图去毛边：选择文字改为无障碍隐藏文本，favicon 加载失败时显示内置网页图标，避免卡片顶部出现破图和多余文字。
- [x] 分页与侧栏状态降权：分页条改为更轻的辅助栏，侧栏状态卡弱化卡片感并沉到侧栏底部，减少对集合树和标签的干扰。

## 建议实施顺序

1. Sprint A：`UI-AUD-001~007`，先停止 CSS 继续互相覆盖。
2. Sprint B：`UI-AUD-101~110`，收口主工作台与移动端核心路径。
3. Sprint C：`UI-AUD-201~208`，重做设置和任务结果展示。
4. Sprint D：`UI-AUD-301~505`，统一外围页面、扩展和可访问性。
5. Sprint E：`UI-AUD-601~605`，模块化、主题和性能治理。

## 完成定义（DoD）

- 对应任务有修改前/后截图或录屏。
- 四档视口无横向溢出、遮挡、不可达操作或双滚动。
- 键盘可完成该功能主流程，焦点不会丢失。
- loading/empty/error/success 均有明确状态。
- `npm run cf:check`、`npm run cf:smoke` 和远端 smoke 通过。
- 若修改核心布局或组件，必须更新视觉回归基线和前端回归清单。
